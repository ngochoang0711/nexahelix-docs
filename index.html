01 / OVERVIEW

What NexaHelix is

A single workspace for business analysts and engineers to turn unstructured requirements documents into a queryable, contradiction-checked, traceable knowledge base — with an AI pipeline doing the reading, and a human doing the deciding.

Enterprise migrations (ERP, WMS, CRM) drown in requirements scattered across RFPs, Confluence spaces, Jira backlogs, and vendor PDFs. Nobody can say with confidence which document is authoritative, which requirements contradict each other, or which specification a given clause actually produced. NexaHelix ingests every source once, compiles it into a typed knowledge graph shared between prose and diagram, and routes every disagreement to a human before it reaches a specification — so the artefact you ship always knows exactly where it came from.

03 / MODULES

The seven surfaces

Each module is a stop along the same pipeline — from raw file to signed-off specification — with its own view into the underlying graph.

MODULE 01 · INGEST
Document Ingestion Hub
SHA-256 · MIME validated

Upload, connect, and govern every source that feeds the knowledge graph. Nothing enters the AI pipeline without review.

  • Drag-and-drop plus live connectors for Jira, Confluence, and Azure DevOps (OAuth 2.0)
  • Five-stage pipeline per file: validation → OCR extract → anonymization → chunking → graph index
  • A governance rule table masks PII before any text reaches an LLM call — not after
  • Full ingestion audit log: tokens, chunks, PII hits, and duration per source
governance · pii rules applied before any llm call

MODULE 02 · KNOWLEDGE BASE
Knowledge Base Explorer
wiki ⇄ graph

A synchronized wiki and graph. Every entity in prose maps to a node — and back.

MODULE 03 · GRAPH
Graph Visualizer
NL or Cypher

The full project topology. Pan, zoom, filter, and query — by natural language or Cypher.

  • Query bar accepts plain English ("show all systems connected to Order Management") or raw Cypher
  • Four relationship types rendered distinctly: DEPENDS_ON, CONTRADICTS, TRIGGERS, MANAGED_BY
  • Selecting any node opens an inspector with its type, description, edges, and exact source citation
  • An accessible data-table view mirrors the canvas for non-visual navigation

MODULE 05 · CONFLICTS
Contradiction & Validation Report
human-in-the-loop

Triage every conflict the Critic Agent has surfaced — and leave with decisions made.

  • Conflicts are classed Logical, Ambiguity, or Missing link, each with a severity from Low to Critical
  • Every conflict shows both source passages side by side, with the disputed values highlighted
  • The Critic Agent proposes a resolution with its reasoning — accept, hand-edit, or dismiss as false positive
  • Every decision is appended to an audit trail: detection time, escalation time, linked open question

MODULE 06 · SPECIFICATIONS
Specification Generator
User Story · Gherkin · BPMN

From graph context to engineering blueprint — in any standard your team uses.

04 / ARCHITECTURE

What it's built on

The stack referenced throughout the prototype's pipeline, graph, and client layers.

Orchestration
Deterministic orchestrator
A six-step job state machine (parser → ingestion → graph → governance → validator → router) plus a bounded validator ↔ story-writer review loop with loop guard — every hand-off written to the audit log.
Graph store
NetworkX / Neo4j
Pluggable store — in-memory NetworkX by default, Neo4j via NEO4J_URI. Typed ontology nodes (Requirement, BusinessRule, Feature, Stakeholder…) and edges (TRACES_TO, REFINES, IMPLEMENTS, TESTED_BY, OWNED_BY, CONTRADICTS).
Document parsing
pypdf · python-docx · csv
Format-specific parsers split ingested files into typed chunks with section metadata, ahead of PII masking and sensitivity classification.
LLM backend
Tiered: Haiku (T1) → Sonnet (T2)
Fast tier for pattern-confirmable work, escalating on low extraction confidence; deterministic heuristic fallback when no key is set; Confidential content routes to a local model only.
Client
NexaHelixAPI
REST client (get / post / postForm / put) targeting localhost:8765/api/v1 — the seam every module reads and writes through, with timeout handling and an offline demo mode.
Governance
Pre-LLM redaction + ownership
Regex masking (email, phone, IPv4, API keys) before any model call; sensitivity and PII flags stored per graph node; a governance agent materializes OWNED_BY links and deliberately leaves unassigned roles visible.
